2024.09.01-2024.12.31

학점 연계 인턴십

주식회사 드제이

SI 사업부 개발팀

기술 스택

java/Spring, Kotlin

헥사고날 아키텍처를 활용한 대규모 알림 서비스 설계

  • 푸시, 이메일, SMS 등 다양한 알림 채널과의 연동이 필요한 시스템 특성상, 외부 시스템과의 결합도를 낮출 수 있는 헥사고날 아키텍처 채택
  • Kotlin의 데이터 클래스로 알림 템플릿과 메시지를 불변객체로 관리하고, sealed 클래스를 활용해 알림 상태와 타입을 타입 안전하게 처리
  • Port와 Adapter 패턴으로 FCM, KAKAO 등 외부 알림 서비스들의 교체와 추가가 용이한 구조 구현
  • 각 알림 채널별 독립적인 테스트와 배포가 가능하도록 설계하여, 서비스 안정성과 유지보수성 향상

사용자 행동 기반 의사결정을 위한 로그 파이프라인 구축

  • 기술 도입 시 정량적인 의사결정의 필요성을 인식하고, 데이터 기반 의사결정 체계 구축
  • Grafana Loki와 Promtail을 활용해 사용자 행동 패턴과 오류 발생 상황을 실시간으로 수집하는 파이프라인 구현
  • 예시: Virtual Thread vs CompletableFuture 도입 검토 시, 처리량과 리소스 사용률 데이터를 통한 객관적 의사결정 진행

이미지 업로드 성능 최적화

문제상황: 다중 사용자 이미지 업로드시 100개 요청에서 100ms까지 지연 발생

해결방법: CompletableFuture 도입으로 여러 이미지 동시 처리 구현

결과: 응답 시간 100ms에서 52ms로 48% 단축

간접 참조 최적화를 통한 N+1 문제 해결

문제상황: 1,000개 데이터 조회시 40초 소요되는 성능 문제 발생

해결방법: PK 추출 후 IN 쿼리와 Map 자료구조를 활용한 O(1) 시간 복잡도 구현

결과: 평균 응답 시간 40초에서 1.8초로 단축

서비스별 최적 비동기 처리 방식 선정

CompletableFuture의 쓰레드 할당 방식을 분석하고, 대량 처리시 발생할 수 있는 부하를 고려했습니다. JVM의 Context Switching 비용과 I/O 바운드 특성을 고려하여 Virtual Thread 방식 채택을 결정했습니다.

엔티티 연관관계와 성능 최적화의 균형

성능 개선 과정에서 객체 지향 설계와 성능 사이의 트레이드오프를 경험했습니다. 엔티티 설계시 지연로딩을 기본으로 하고 필요한 경우 즉시로딩을 사용하는 방식으로 두 가지 요구사항의 균형을 맞추었습니다.

주요 성장 포인트

• 비동기 처리 방식의 트레이드오프를 고려한 기술 선정 능력 향상

• 명확한 목적을 가진 모니터링 도구 도입의 중요성 인식

• 시스템 응답 시간 개선과 효율적 리소스 관리 능력 향상

• 객체 지향 설계와 성능 사이의 균형점 도출 역량 강화
